Runs OptKnock in the most general form

USAGE

OptKnock (model, selectedRxnList, options, constrOpt, prevSolutions, verbFlag, solutionFileNameTmp)

INPUTS

  • model â Structure containing all necessary variables to described a stoichiometric model

    • rxns - Rxns in the model

    • mets - Metabolites in the model

    • S - Stoichiometric matrix (sparse)

    • b - RHS of Sv = b (usually zeros)

    • c - Objective coefficients

    • lb - Lower bounds for fluxes

    • ub - Upper bounds for fluxes

    • rev - Reversibility of fluxes

  • selectedRxnList â n x 1 cell array where each cell is a reaction in model.rxns that can be knocked-out in OptKnock

OPTIONAL INPUTS

  • options â OptKnock options

    • targetRxn - Target flux to be maximized

    • numDel - # of deletions allowed (Default: 5)

    • numDelSense - Direction of # of deletions constraint (G/E/L) (Default: L)

    • vMax - Max flux (Default: 1000)

    • solveOptKnock - Solve problem within Matlab (Default: true)

    • createGams - Create GAMS input file

    • gamsFile - GAMS input file name

  • constrOpt â Explicitly constrained reaction options

    • rxnList - Reaction list

    • values - Values for constrained reactions

    • sense - Constraint senses for constrained reactions (G/E/L)

  • prevSolutions â Previous solutions

  • verbFlag â Verbose flag

  • solutionFileNameTmp â File name for storing temporary solutions

OUTPUTS

  • optKnockSol â optKnock solution structure This is the same as the solution Structure from solveCobraMILP plus the following fields (if a solution exists): .rxnList - Reaction KO list .fluxes - the flux distribution

  • bilevelMILPproblem â optKnock problem structure

OptKnock uses bounds of -vMax to vMax or 0 to vMax for reversible and irreversible reactions. If you wish to constrain a reaction, use constrOpt.

analyzeGCdesign(modelRed, selectedRxns, target, deletions, maxKOs, objFunction, delPenalty, intermediateSlns)[source]Â¶

Analyzes results with replacement knockouts should get closer to local maxima. Must have num KOs > 1.

USAGE

[improvedRxns, intermediateSlns] = analyzeGCdesign (modelRed, selectedRxns, target, deletions, maxKOs, objFunction, delPenalty, intermediateSlns)

INPUTS

  • modelRed â reduced model

  • selectedRxns â selected reaction list from the reduced model

  • target â exchange rxn to optimize

  • deletions â initial set of KO rxns (must have at least 1 rxn)

OPTIONAL INPUTS

  • maxKOs â maximum number of rxn KOs to allow (Default = 10)

  • objFunction â pick an objective function to use (Default = 1):

    1. obj = maxRate (yield)

    2. obj = growth*maxRate (SSP)

    3. obj = maxRate*(delPenalty^numDels) (yield with KO penalty)

    4. obj = growth*maxRate*(delPenalty^numDels) (SSP with KO penalty)

    5. obj = maxRate*(slope^(-1)) (GC_yield)

    6. obj = growth*maxRate*(slope^(-1)) (GC_SSP)

    7. obj = maxRate*(delPenalty^numDels)*(slope^(-1)) (GC_yield with KO penalty)

    8. obj = growth*maxRate*(delPenalty^numDels)*(slope^(-1)) (GC_SSP with KO penalty)

  • delPenalty â penalty on extra rxn deletions (Default = .99)

  • intermediateSlns â Previous set of solutions (Default = deletions)

OUTPUTS

  • improvedRxns â the KO rxns for an improved strain

  • intermediateSlns â all the sets of best KO rxns that are picked before the final set is reached

optGene(model, targetRxn, substrateRxn, generxnList, varargin)[source]Â¶

Implements the optgene algorithm.

USAGE

[x, population, scores, optGeneSol] = optGene (model, targetRxn, substrateRxn, generxnList, MaxKOs, population)

INPUTS

  • model â Model of reconstruction

  • targetRxn â (char) String name of reaction which is to be maximized

  • substrateRxn â (char) Substrate reactions

  • generxnList â (cell array) List of genes or rxns which can be knocked out. The program will guess which of the two it is, based on the content in model.

OPTIONAL INPUTS

  • MaxKOs â (double) Maximal KnockOuts

  • population â (logical matrix) population matrix. Use this parameter to interrupt simulation and resume afterwards.

  • mutationRate â (double) the rate of mutation.

  • crossovermutationRate â (double) the rate of mutation after a crossover. This value should probably be fairly low. It is only there to ensure that not every member of the population ends up with the same genotype.

  • CrossoverFraction â (double) Percentage of offspring created by crossing over (as opposed to mutation). 0.7 - 0.8 were found to generate the highest mean, but this can be adjusted.

  • PopulationSize â (double) Number of individuals

  • Generations â (double) Maximum number of generations

  • TimeLimit â (double) global time limit in seconds

  • StallTimeLimit â (double) Stall time limit (terminate after this much time of not finding an improvement in fitness)

  • StallGenLimit â (double) terminate after this many generations of not finding an improvement

  • MigrationFraction â (double). how many individuals migrate

  • MigrationInterval â (double). how often individuals migrate from one population to another.

  • saveFile â (double or boolean) saving a file with inputs and outputs. Default = false;

  • outputFolder â (char) name of folder where files will be generated

OUTPUTS

  • x â best optimized value found

  • population â Population of individuals. Pass this back into optgene to continue simulating where you left off.

  • scores â An array of scores

  • optGeneSol â optGene solution strcture
